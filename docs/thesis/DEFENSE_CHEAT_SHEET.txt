  3. FORMULAS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Per-Image:
    Precision = TP / (TP + FP)
    Recall    = TP / (TP + FN)
    F1        = 2 x (Prec x Rec) / (Prec + Rec)
    Accuracy  = TP / (TP + FP + FN)          [Jaccard Index]

  Validator Average (macro):
    Validator_Metric = (1/n) x SUM(Image_Metric_i)    for n images

  Grand Average:
    Grand = (VAL1_avg + VAL2_avg + VAL3_avg) / 3

  WORKED EXAMPLE — VAL1, Image 1:
    Ground truth = 10 mealybugs
    Detections = 6 boxes (all correct, 0 false)
    TP=6  FP=0  FN=10-6=4

    Precision = 6/(6+0) = 100.0%
    Recall    = 6/(6+4) = 60.0%
    F1        = 2x(1.0x0.6)/(1.0+0.6) = 75.0%
    Accuracy  = 6/(6+0+4) = 60.0%

  4. HARD QUESTIONS & ANSWERS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Q: "61% mAP is low. Other studies report 90%+."
A: Those detect larger objects (fruits, leaves) with large models. Mealybugs are
   10-30px, clustered, semi-transparent, on a 5.4MB Nano model. Published
   small-pest benchmarks (Song et al. 2025) report 45-65% mAP. Our 61% is at
   the TOP of that range. Expert validation confirms 91.75% F1 in practice.

───────────────────────────────────────────────────────────────────────────────

Q: "Data leakage — v10 test images may be in v13afix training. Invalid?"
A: Acknowledged in limitations. v10 test (462) is 3.4% of v13afix training
   (13,664) — too small to inflate mAP by 12.9pp. Proof: the guaranteed-clean
   native test (1,952 imgs) shows 61.0% — HIGHER than the potentially-leaked
   set (56.7%). If leakage helped, leaked set would score higher, not lower.

───────────────────────────────────────────────────────────────────────────────

Q: "Only 3 validators, 21 images — statistically too weak."
A: 95% CI for F1: 91.75% +/- 5.15% = [86.6%, 96.9%]. We don't rely solely on
   this — it complements the 1,952-image automated benchmark. Validators are
   domain experts (OMAG staff). 7 scenarios cover key environmental variations.
   This is cross-validation of operational fitness, not the primary metric.

───────────────────────────────────────────────────────────────────────────────

Q: "No architecture comparison (MobileNet, EfficientDet). How do you know
    YOLO26n is best?"
A: Compared YOLO26n vs YOLO11n (65.1% vs 52.6%, same data). Full multi-arch
   ablation infeasible given compute/time constraints. Selection driven by:
   (1) TFLite export compatibility, (2) <300ms on budget hardware, (3) YOLO
   consistently outperforms SSD/EfficientDet on small pests in literature.
   Acknowledged as future work in Section 5.1.

───────────────────────────────────────────────────────────────────────────────

Q: "How do you know the model converged? No hyperparameter tuning?"
A: Early stopping triggered at epoch 75 (patience=15, best at ~epoch 60).
   Val loss plateaued. Used Ultralytics defaults (community-validated on COCO).
   Our improvements came from DATA quality (cleaning, field images), not HP
   tuning — proven by v10->v13afix gains from dataset changes alone.

───────────────────────────────────────────────────────────────────────────────

Q: "Precision 99% but recall only 60%. Problem?"
A: DELIBERATE design choice. False positive = farmer applies unnecessary
   pesticide (economic + environmental harm). Missed detection = farmer
   rechecks later (lower cost). We explicitly traded recall for precision via
   the 30% threshold. This follows "first, do no harm" for advisory tools.

───────────────────────────────────────────────────────────────────────────────

Q: "Why conf=0.12 for benchmark but 0.30 for deploy?"
A: Different purposes. 0.12 = Ultralytics default for computing full P-R curve
   (enables fair comparison with literature). 0.30 = operational filter tuned
   for near-zero false positives in practice. One measures CAPABILITY, the
   other controls USER EXPERIENCE.

───────────────────────────────────────────────────────────────────────────────

Q: "IoU=0.5 is too strict for tiny objects."
A: Valid point — this is why mAP@0.5:0.95 is only 29.2%. We report both.
   mAP@0.5 is the community standard for comparability. Expert validation
   uses lenient "is box on bug?" criteria → 91.75% F1. Future work: report
   mAP@0.25 for small-pest-specific evaluation.

───────────────────────────────────────────────────────────────────────────────

Q: "What if farmer takes blurry/dark photo?"
A: Model produces lower confidence → filtered by 30% threshold → fewer/no
   detections shown. App includes capture guidance. Offline design allows
   immediate retake. Average confidence of 57% in field testing shows model
   is appropriately cautious under real conditions.

───────────────────────────────────────────────────────────────────────────────

Q: "Modified waterfall is wrong for ML. Should be agile."
A: Waterfall governs THESIS deliverables (proposal→design→implement→test→
   defense). ML pipeline was iterative (12 model versions with data-driven
   feedback loops). "Modified" = sequential thesis milestones + iterative
   technical development. Standard approach for undergraduate CS theses.

───────────────────────────────────────────────────────────────────────────────

Q: "40% miss rate could harm farmers. What about liability?"
A: App is a DETECTION AID, not diagnostic replacement. Like a medical
   screening tool — high specificity means flagged areas reliably need
   attention. App does NOT recommend treatment; it flags presence for
   farmer's judgment. 60% detection in one photo pass still provides
   actionable intelligence they lack without the tool.

───────────────────────────────────────────────────────────────────────────────

Q: "Your 'Accuracy' is not standard accuracy. Misleading?"
A: It's the Jaccard Index. TN is infinite in object detection (every empty
   pixel = TN), making classical accuracy meaningless (~100% always).
   Padilla et al. 2020, Everingham et al. 2010 establish this. We can
   relabel as "Detection Accuracy (Jaccard)" if preferred.

───────────────────────────────────────────────────────────────────────────────

Q: "Inter-rater reliability? Cohen's Kappa?"
A: Not applicable — each validator assessed DIFFERENT images (7 unique each),
   not the same images. Kappa/Alpha require multiple raters on the SAME items.
   The consistency in precision (98.75%-100%) across different images and
   validators demonstrates reliable near-zero false positive rate.

───────────────────────────────────────────────────────────────────────────────

Q: "This isn't novel. Pest detection apps exist."
A: Novelty is the INTEGRATED system: offline-first mobile detection +
   geospatial mapping + severity scoring, specifically for Philippine
   pineapple mealybugs. No existing tool serves this specific agricultural
   context with offline capability and local-only data storage.

───────────────────────────────────────────────────────────────────────────────

Q: "Why not just use a pretrained model from the internet?"
A: Pretrained COCO/ImageNet models don't include mealybug classes. No
   public mealybug detection model exists. Fine-tuning on domain-specific
   data is mandatory. Our 12-iteration improvement (21%→61%) demonstrates
   the necessity of custom training.
